    Aufgabe 1 - Einkaufsliste (Link)

    Dauer: 30 - 45 Minuten

    In dieser Aufgabe geht es darum, die Applikationslogik einer bestehenden HTML-Seite zu erstellen. Dabei handelt es sich um eine Einkaufsliste, der über ein Input-Feld neue Einträge hinzugefügt werden können. Hinzugefügte Einträge wandern in die Einkaufsliste. Ein Klick auf einen der Artikel markiert diesen als erledigt (CSS-Klasse "ok"). Klickt man erneut auf den Artikel wird die Markierung wieder rückgängig gemacht. Über den "Liste leeren"-Button werden alle Einträge gelöscht.

    Regeln:

    • Der Code muss sich in der "app.js" befinden
    • jQuery ist eingebunden und darf benutzt werden
    • Die index.html darf nicht bearbeitet werden
    • Es dürfen keine weiteren Dateien erstellt werden
    • "Don't pollute the global namespace!"

    Entwicklung:

    Für diese Aufgabe benötigen wir keinen Webserver. Die "index.html" kann also direkt vom Filesystem im Browser geöffnet werden

    Information / Hilfestellung / Hinweise:

    • http://jkoder.com/avoid-polluting-global-namespace-in-javascript-iifeimmediately-invoked-function-expression/
    • http://gregfranko.com/jquery-best-practices/
    • http://api.jquery.com/

    Themengebiete der Aufgabe:

    jQuery, DOM-Manipulation, IIFE, global namespace

    Aufgabe 2 - Einkaufsliste Part2 (Link)

    Dauer: 30 - 45 Minuten

    Diese Aufgabe baut auf die vorhergehende Aufgabe auf. Du kannst also deine Lösung für Aufgabe1 in die "app.js" Datei kopieren. Um unsere Einkaufsleiste zu verbessern möchten wir nun das Speichern der Daten implementieren. Der Webbrowser bietet mit dem HTML5 Standard 2 Storage-Möglichkeiten die es erlauben Daten auf Seiten des Clients zu speichern. Ziel dieser Aufgabe ist es eine der 2 Storage-Möglichkeiten zu wählen und anschließend das Speichern und Laden der Daten zu implementieren. Dabei bietet es sich an, die einzelnen Objekte (Models) als Klasse zu definieren.

    • Eine Einkaufsliste enthält mehrere Artikel
    • Ein Artikel besitzt einen Namen und einen Zustand, der darauf schließen lässt, ob der Artikel abgehakt ist
    • Alle Objekte (Models) sollen über ein Erstellungs- und Bearbeitungsdatum verfügen

    Regeln:

    • Der Code muss sich in der "app.js" befinden
    • jQuery ist eingebunden und darf benutzt werden
    • Die index.html darf nicht bearbeitet werden
    • Es dürfen keine weiteren Dateien erstellt werden
    • "Don't pollute the global namespace!"
    • Die Daten (Models) sollen als Klasse definiert werden

    Entwicklung:

    Für diese Aufgabe benötigen wir keinen Webserver. Die "index.html" kann also direkt vom Filesystem im Browser geöffnet werden

    Information / Hilfestellung / Hinweise:

    • https://developer.mozilla.org/de/docs/Web/JavaScript/Introduction_to_Object-Oriented_JavaScript
    • https://developer.mozilla.org/en-US/docs/Web/API/Web_Storage_API
    • https://developer.mozilla.org/en-US/docs/Web/API/Web_Storage_API/Using_the_Web_Storage_API

    Themengebiete der Aufgabe:

    Objekt Orientierung, Klassen, Vererbung, Prototypal Inheritance, Local / Session Storage

    Aufgabe 3 - Einkaufsliste Part3 (Link)

    Dauer: 45 Minuten - 1 Stunde

    Diese Aufgabe baut auf die vorhergehende Aufgabe auf. Du kannst also deine Lösung für Aufgabe2 in die "app.js" Datei kopieren. Jetzt ist es an der Zeit unsere kleine Applikation zu refactoren. Bisher ist unser Code in der "app.js" zu finden. Allerdings macht es Sinn, die logischen Bestandteile in einzelne Teile (= Module) zu zerlegen und in jeweils eigene Dateien auszulagern. Ziel dieser Aufgabe ist es den bestehenden Code der "app.js" in einzelne Module aufzuteilen und diese als "bundle.js" an den Browser auszuliefern. Zum lösen dieses Problems gibt es mehrere Tools. An dieser Stelle wollen wir uns zunächst mit Browserify beschäftigen

    Regeln:

    • Der Code muss sich letztlich in der "bundle.js" befinden
    • Die index.html darf nicht bearbeitet werden
    • Der Code der "app.js" soll in einzelne Module aufgeteilt werden

    Entwicklung:

    Für diese Aufgabe benötigen wir keinen Webserver. Die "index.html" kann also direkt vom Filesystem im Browser geöffnet werden

    Information / Hilfestellung / Hinweise:

    • http://browserify.org/
    • https://petecoop.co.uk/blog/modular-javascript-with-browserify

    Themengebiete der Aufgabe:

    Module, Refactoring, Browserify
